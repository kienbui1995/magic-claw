Capabilities
Everything you need to run an AI workforce
Nine modules that handle registration, routing, orchestration, cost tracking, and more.
🔍

Smart Routing

Route tasks to the best worker by capability match, cost, speed, or custom strategy. Overloaded workers are automatically skipped.

📊

Cost Control

Track spending per worker and team. Get alerts at 80% budget. Workers auto-pause at 100%. No more surprise bills.

🔀

DAG Workflows

Multi-step pipelines with parallel execution, dependency resolution, and failure handling (retry, skip, abort).

✅

Output Validation

JSON schema validation on every task output. Bad results get caught before they reach your users.

👥

Team Management

Organize workers into teams with budgets and policies. Assign, remove, and track workers across your organization.

📖

Knowledge Hub

Shared knowledge base with full-text search. Workers access the same context — no duplicate RAG queries.

🛡

Security

API key auth, SSRF protection, body size limits, configurable CORS, security headers, graceful shutdown.

🔄

Retry & Recovery

Failed dispatches retry with backoff. Workers report heartbeats. Offline workers get detected and removed.

✋

Human-in-the-Loop

Approval gates in workflows. Critical steps pause for human review before proceeding.

Python SDK
A worker in 10 lines
Any Python function becomes a managed AI worker. Register capabilities, connect to the server, start serving. That's it.

Works with CrewAI, LangChain, or plain Python.
Your agents. Your framework. MagiC manages.

from magic_ai_sdk import Worker worker = Worker("ContentBot", "http://localhost:9000") @worker.capability("summarize") def summarize(text: str) -> str: # Your logic here (LLM, ML, or plain Python) return f"Summary of: {text[:50]}..." worker.register("http://localhost:8080") worker.serve()
